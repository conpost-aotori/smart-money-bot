# Smart Money Daily Snapshot Bot

Posts a 1200×720 PNG card to **Discord** and **X** three times per day
summarising the last 24 hours on **Hyperliquid** perps:

- 🟢 **Top buys (24h)** — coins that ripped (longs winning)
- 🔴 **Top sells (24h)** — coins that puked (shorts winning)
- 🔥 **Sectors heating** — 24h notional volume aggregated by sector tag

Data comes straight from Hyperliquid's public ``/info`` endpoint (no API
key required). Japanese labels are auto-generated by Gemini and cached in
SQLite, so each ticker is translated once and reused.

> **Built by remixing [VirtualNISHI/polymarket-BOT](https://github.com/VirtualNISHI/polymarket-BOT)**.
> The daily-snapshot stack (Discord+X poster, image card renderer,
> Gemini-backed Japanese label translator, GitHub Actions cron) is
> reusable; only the data-collection layer is domain-specific.

## Architecture

```
[GitHub Actions cron, 3× JST]
        ↓
  [Restore SQLite from `bot-state` branch]
        ↓
  [Hyperliquid /info metaAndAssetCtxs]   ← single POST, no auth
        ↓
  [collector.collect_snapshot]            ← filter by 24h volume
        ↓
  [top_buys / top_sells / sectors_heating]
        ↓
  [Gemini translator]                     ← coin → JP label, cached
        ↓
  [render_snapshot_png]                   ← Pillow + pilmoji + Noto CJK
        ↓
  [Discord webhook (image)]   [X media_upload + create_tweet (image)]
        ↓
  [Persist SQLite to `bot-state` branch]
```

Per-run cost: 1 HL POST + ~6 short Gemini translations (free tier) + 1
Discord POST + 1 X media upload + 1 tweet POST. Total wall time ~30s on
``ubuntu-latest``.

## Local setup

```bash
python3.11 -m venv .venv
source .venv/bin/activate         # Windows: .venv\Scripts\activate
pip install -e .

cp .env.example .env
# fill in DISCORD_WEBHOOK_URL, X_*, GEMINI_API_KEY

python -m src.db init             # create SQLite schema
python scripts/run_daily.py --dry-run
```

`--dry-run` builds the image and tweet text but doesn't post.

## GitHub Actions secrets

Set in **repo → Settings → Secrets and variables → Actions**:

| Secret | Required | Notes |
|---|---|---|
| `DISCORD_WEBHOOK_URL` | ✓ | Channel webhook |
| `DAILY_SNAPSHOT_DISCORD_WEBHOOK_URL` |  | Optional override; falls back to `DISCORD_WEBHOOK_URL` |
| `X_API_KEY` | ✓ | Consumer Key (25 chars) |
| `X_API_SECRET` | ✓ | Consumer Key Secret (50 chars) |
| `X_ACCESS_TOKEN` | ✓ | OAuth 1.0a User Access Token |
| `X_ACCESS_SECRET` | ✓ | OAuth 1.0a User Access Token Secret |
| `GEMINI_API_KEY` |  | Optional; without it labels fall back to raw tickers |
| `ANTHROPIC_API_KEY` |  | Optional alternate translator |

## Schedule

`5 15,23,7 * * *` UTC = **00:05 / 08:05 / 16:05 JST**. Edit
``.github/workflows/daily-snapshot.yml`` to change cadence.

## Customizing

- **Sector buckets** live in ``config/settings.yaml`` → ``sector_map``.
  Add a coin under whatever label fits; first match wins.
- **Tweet hashtags** live in ``src/daily_snapshot/formatter.py`` →
  ``build_tweet`` default ``hashtags`` arg.
- **Image colors** live at the top of
  ``src/daily_snapshot/image_renderer.py``.
- **Card sections** are defined in two places (kept in sync manually):
  ``image_renderer.py`` (``sections`` tuple) and ``job.py`` (the three
  ``top_buys / top_sells / sectors_heating`` calls + the corresponding
  ``_persist`` calls).
